# Universal Writing Agent-Kit — エージェント運用マニュアル (AGENTS.md)

これは、様々な媒体で高品質なテキストを生成するための **AIライティング＆コンテンツ制作プラグイン** です。特化型エージェント、用途別のスキル、ワークフロー、および自動化された校正フックを提供します。

## コア原則（Core Principles）

1. **エージェント・ファースト (Agent-First)** — 企画、調査、執筆、推敲の各工程を、専門に特化したエージェントに委譲します。
2. **パターン駆動 (Pattern-Driven)** — トーン、スタイル、構成は、選択されたパターン（記事、小説、論文など）に基づいて変化します。
3. **事実最優先 (Fact-First)** — すべての主張、統計、引用を検証し、決して事実を捏造（ハルシネーション）しないでください。
4. **執筆と推敲の分離 (Draft & Polish)** — 「書く行為」と「編集する行為」は常に分離して行います。
5. **実行前の計画 (Plan Before Execute)** — 複雑な文書や長文を作成する場合は、最初の1文字を書く前に必ず全体の構成を計画します。

## 利用可能なエージェント (Available Agents)

| エージェント名 | 目的・役割 | いつ使用するか |
|-------|---------|-------------|
| `content-director` | 企画立案、構成作成、ディレクション | あらゆる執筆プロジェクトの立ち上げ時 |
| `researcher` | 事実収集、Web検索、コンテキスト構築 | 執筆を開始する前（エビデンスが必要な場合） |
| `drafter` | 初稿（テキスト）の生成 | 構成案に基づいて文章を書き起こす時 |
| `copy-editor` | 文章の洗練、文法の修正、トーンの確認 | 初稿（ドラフト）が完成した後 |
| `fact-checker` | 主張と論理の検証 | 最終化する前（特にノンフィクションの場合） |

*注: 特定のタスクに特化するため、上書きルールを持つエージェント（例: `patterns/article/drafter.md`）も存在します。*

## 利用可能なパターン群 (MDCルール)

基本ルール (`00-writing-common.mdc`) は常に適用されます。`content-director`は、タスクに応じて以下の追加パターンを割り当てます：

| パターン名 | 対象ドキュメント | 特徴・ルール |
|---------|---------|-------------|
| `01-pattern-article.mdc` | Web記事、ブログ、コラム | SEO意識、読みやすさ重視、拾い読みしやすい構成 |
| `02-pattern-novel.mdc` | 小説、物語、台本、脚本 | 創作重視、「語らずに見せる」、会話文と情景描写 |
| `03-pattern-academic.mdc` | 学術論文、公式文書、マニュアル | 厳密な客観性、引用の必須化、構造化されたフォーマット |

## エージェントのオーケストレーション（連携・指揮）

用途に合わせたワークフローを通じて、エージェントを自律的に連携させます：
- **複雑・長文のドキュメント** → **long-form-generation**（長文生成）ワークフローを使用 (`content-director` -> ユーザー承認 -> `drafter` の反復ループ)
- **メールやSlackの迅速な返信** → **rapid-response**（迅速対応）ワークフローを使用 (ディレクターを飛ばし、直接 `drafter` にコンテキストを渡して生成)
- **既存コンテンツの流用・再利用** → **repurposing**（コンテンツ変換）ワークフローを使用 (`content-director` が複数フォーマットを定義し、`drafter` が並行処理)

## 検証とフック (Verification & Hooks)

**最終化（Pre-Commit）を行う前の必須プロセス：**
- フォーマット検証フック（例：`hooks/pre-commit-text.js`）を実行します。
- テキスト内に禁止されているAI特有のクリシェ（例：「結論として」「〜のタペストリーが」）が含まれていないことを確認します。
- フォーマットの整合性（Markdownの見出し階層、リンク等）を検証します。

**事実誤認（エラー）が発見された場合:** 進行を停止（STOP） → `fact-checker`エージェントを使用 → 致命的（CRITICAL）な主張を修正 → 類似のハルシネーションがないか、ソース元のリサーチを再確認。

## 執筆スタイルと品質

**ライティングの質 (CRITICAL):**
決して汎用的なAIの「穴埋めテキスト」に頼らないでください。要求されたトーン（フォーマル、親しみやすい、会話調、学術的など）を厳格に順守してください。

**プロセス・チェックリスト:**
- ドキュメントの構成案（アウトライン）は作成されたか？ (Yes/No)
- 事実データは収集され、検証されたか？ (Yes/No)
- ドラフトは指定されたパターンのMDCルールに準拠しているか？ (Yes/No)
- 文脈の流れと文法は推敲（コピーエディット）されたか？ (Yes/No)

## プロジェクト構造（ディレクトリ構成）

```
.cursor/rules/    — 普遍的な執筆ルールとパターン固有のガイドラインを定義するMDCルール群
agents/           — 特化型の執筆サプエージェント群（共通エージェント & パターン上書き）
  common/         — 汎用エージェント（researcher, drafter, copy-editor, fact-checker, content-director）
  learning/       — CFO学習専用エージェント（kb-navigator, quiz-master, scenario-coach）
skills/           — フォーマット別のアダプター（ビジネス、クリエイティブ、学術、SNS/PRなど）
                    + CFO学習スキル（cfo-quiz, kb-lookup, week-review, scenario）
workflows/        — 複数エージェントによる実行プレイブック（長文、迅速、再利用・変換）
                    + CFO学習ワークフロー（weekly-study, kb-deep-dive）
hooks/            — コミット・最終化前のテキスト検証スクリプト
kb/               — freee 知識ベース（2,424記事、36カテゴリ）
roadmap/          — CFO学習計画・週次ガイド・学習ログ
```

## CFO学習補助ツール

freee KB（2,424記事）を活用したCFO向け16週間学習カリキュラムをサポートする専用ツール群です。

### 学習専用エージェント

| エージェント名 | 目的・役割 | いつ使用するか |
|---|---|---|
| `kb-navigator` | KB記事の探索・優先度付き提示 | トピックの調査、Week記事の特定 |
| `quiz-master` | 理解度クイズの生成と採点 | 学習後の確認、苦手範囲の特定 |
| `scenario-coach` | CFOシナリオの生成とコーチング | 知識を判断力に変換するトレーニング |

### 学習スキル（スラッシュコマンド）

| コマンド | 説明 | 使用例 |
|---|---|---|
| `/cfo-quiz` | Week/トピックのクイズを生成 | `/cfo-quiz Week 5` |
| `/kb-lookup` | KB検索・用語解説 | `/kb-lookup 定期同額給与` |
| `/week-review` | 週次振り返り・ログ記録支援 | `/week-review Week 3` |
| `/scenario` | CFOビジネスシナリオ実践 | `/scenario 資金繰り 実戦` |

### 学習ワークフロー

| ワークフロー | 説明 | 所要時間 |
|---|---|---|
| `weekly-study` | 1週分の学習を完結（KB→クイズ→シナリオ→ログ） | 60〜90分 |
| `kb-deep-dive` | 1トピックを体系的に深掘り（要点シート生成付き） | 40〜70分 |

### 典型的な学習フロー

```
# 通常の学習（1セッション）
weekly-study Week {N}
  └─ Step 1: kb-navigator が記事を特定
  └─ Step 2: 精読サポート
  └─ Step 3: quiz-master がクイズ生成
  └─ Step 4: scenario-coach がシナリオ実践
  └─ Step 5: ログ記録

# 用語が分からないとき
/kb-lookup {用語}

# もっと深く知りたいとき
kb-deep-dive {トピック}
  └─ KB全量サーベイ → 構造化インプット → クイズ → シナリオ → 要点シート保存

# 週末の振り返り
/week-review Week {N}  → /cfo-quiz Week {N}
```

## 成功の定義 (Success Metrics)

- ノンフィクションにおける事実のハルシネーションが **ゼロ** であること。
- トーンが要求されたパターンに **完全に一致** していること（小説にビジネス用語が混ざらない、学術論文に俗語が混ざらない）。
- 高い可読性と、自然なナラティブ（物語・論理）の流れがあること。
- 禁止されたAIクリシェ（使い古された表現）が **ゼロ** であること。
