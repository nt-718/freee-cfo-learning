# レジュメ作成スキル（freee KB → 学習ハンドアウト）

ユーザーが「レジュメを作成して」「Day Xのレジュメを作って」と依頼した場合にこのスキルを使用する。

---

## ステップ1：ロードマップと対象記事の確認

1. `/Users/nt718/learning/freee/roadmap/` 以下のロードマップファイルを読み、対象Dayのテーマ・参照KB記事を確認する
2. 既存レジュメ（`/Users/nt718/learning/freee/roadmap/resumes/`）を確認し、Day番号・ファイル名の命名規則を把握する

**ファイル命名規則**: `week{週番号2桁}_day{日番号2桁}_{テーマ}.md`
- 例: `week01_day03_法人の種類を比較する.md`

---

## ステップ2：KB記事の読み込み

参照すべきKB記事を**並列で同時読み込み**する（効率化のため）。

**KB記事のパス構造**:
```
/Users/nt718/learning/freee/kb/「会社設立」の基礎知識/{カテゴリ番号}_{カテゴリ名}/{記事番号}_{タイトル}.md
```

**読み込み時の注意点**:
- 1つのDayに複数記事がある場合は並列読み込みで時間を節約する
- 記事から「重要数字」「比較表」「箇条書き構造」を優先的に抽出する
- CFO・財務的観点で特に重要な部分（税率、費用、リスク）に注目する

---

## ステップ3：レジュメの構成（テンプレート）

以下の構成を**必ず守る**。セクション順序・見出しレベルを変更しない。

```markdown
# Day {N} レジュメ｜{テーマタイトル}

> **{Week名} — {週テーマ}** | Day {範囲}: {セクションテーマ}

---

## 今日のゴール

---

## 1. {最初の大テーマ}

### {サブセクション}

{内容 — 表・箇条書き・コードブロックを積極活用}

---

## 2. {次のテーマ}

...（記事内容に応じてセクションを追加）

---

## CFO目線の Key Insight

> **{インサイトのタイトル（太字・命令形）}**
>
> {3〜5文。CFO・財務責任者の視点から「なぜこの知識が経営判断に重要か」を述べる。
> 数字・リスク・機会コストを必ず含める。抽象論より具体的な判断基準を示す。}

---

## 今日の振り返りチェック

- [ ] {確認事項1（「〜説明できるか？」形式）}
- [ ] {確認事項2}
- [ ] {確認事項3}
- [ ] {確認事項4}

{次のDayで読む記事名と難易度（★★★/★★/★）を記載}

---

*参照記事: `kb/{カテゴリ}/{記事ID}` ...*
```

## ステップ4：各セクションの作成指針

### 「今日のゴール」
- 箇条書きではなく1〜2文の散文で書く
- 「〜を理解し、〜できるようになる」の形式を基本とする

### テーマセクション（## 1. 〜）
- **比較がある場合は必ず表（Markdown table）で表現する**
- 重要な数字は**太字**にする（例：**22.2万円**、**800万円**）
- 3つ以上の項目は箇条書きより表の方が見やすい
- コードブロック（```）は構造・フロー・階層関係の表現に使う

### 「CFO目線の Key Insight」
- **経営判断・意思決定のフレームを提示する**（単なる知識整理ではない）
- 「〜するな」「〜で判断せよ」「〜から逆算する」等の命令形で言い切る
- 具体的な数字・比較・タイミングを必ず入れる
- テーマに応じた視点：税務コスト・資金調達・リスク管理・事業継続性・機会コスト

### 「振り返りチェック」
- 4〜6項目が適切（少なすぎず多すぎず）
- `- [ ]` 形式（チェックボックス）で統一
- 「〜できるか？」「〜を言えるか？」「〜説明できるか？」で締める

### Day 7（週末振り返り）の特別構成
Week最終日（Day 7/14/21...）は通常のテーマレジュメではなく振り返りシートにする：

```markdown
## Week N 全体マップ（概観図）
## 1. 重要知識チェックリスト（週全体を網羅）
## 2. CFO視点：Week N の核心N原則
## 3. アウトプット課題（3問：基礎確認・比較分析・数値計算）
## 4. Week N の重要数字まとめ（表形式）
## 5. 自己評価（◎/○/△の3段階）
## 次週（Week N+1）の予告
```

## ステップ5：品質チェック

作成後に以下を確認する：

- [ ] 「今日のゴール」が具体的で測定可能な学習目標になっているか
- [ ] 比較が必要な箇所はすべて表で表現されているか
- [ ] **重要な数字**がすべて太字になっているか
- [ ] CFO Key Insightが「経営判断の視点」を提供しているか（知識のまとめになっていないか）
- [ ] 振り返りチェックが「説明できるか」レベルで問えているか（単なる「知っているか」ではない）
- [ ] 参照記事のパスが正確に記載されているか
- [ ] ファイル名の命名規則が正しいか
