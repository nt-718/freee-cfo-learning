# Long-Form Generation Workflow (長文生成ワークフロー)

ユーザーが1回の出力生成に収まりきらない長さのドキュメント（例：10,000語の物語、50ページのビジネスレポート、または100項目のリスト）を要求した場合にこのワークフローを使用してください。

## ステップ

1. **戦略的プランニング (`content-director`)**
   - `content-director` (企画ディレクター) がプロンプトを分析し、包括的な章（チャプター）ごとの構成案を作成します。
   - **ユーザーの承認を待つ**: 次に進む前に、作成した構成案をユーザーに確認・承認してもらいます。

2. **順次（直列）執筆 (`drafter`)**
   - ループ処理を実行します。`drafter`（ドラフター）が構成案に基づいて「第1章」を執筆します。
   - 連続性（コンティニュイティ）を維持する必要がある場合は、`drafter` に対して以前の章の正確なコンテキスト（文脈）を提供（プロンプト）してください。
   - 各章を一時的なMarkdownファイル（例：`chapter_1.md`）として保存します。

3. **段階的レビュー (`copy-editor` & `fact-checker`)**
   - 新しく起草された章に対して `copy-editor`（推敲担当）を実行します。
   - その章に対して `fact-checker`（事実確認担当）を実行します。
   - 修正を適用し、最終化された章を保存します。

4. **ステップ2と3の繰り返し**
   - 構成案がすべて完了するまで、次の章に進みます。

5. **最終的な統合（コンパイル）**
   - すべての章のMarkdownファイルを1つのマスタードキュメントに結合します。
   - 最後に、全体の一貫性を確認するためのグローバルレビューを1回実行します。
