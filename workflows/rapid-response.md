# Rapid Response Workflow (迅速対応・短文ワークフロー)

ユーザーがメール、Slackのスレッド、または短いコミュニケーションに対して即座の返信を必要とする場合にこのワークフローを使用してください。スピードを優先するため、重いプランニング（企画ディレクター）のエージェントはスキップします。

## ステップ

1. **コンテキスト分析 (`drafter`)**
   - `drafter` (執筆者) が元のメッセージ（返信するスレッドやメール）を確認します。
   - 核となる質問（何に対する返信か）、または必要な確認事項を特定します。

2. **迅速な実行 (`drafter`)**
   - `drafter` が簡潔で丁寧な返信を生成します。
   - `content-director` による構成案は必要ありません。
   - 出力の長さは媒体に合わせてください（Slackの場合は1〜2文、メールの場合は1〜2段落）。

3. **トーンの検証 (`copy-editor`)**
   - `copy-editor`（推敲担当）による迅速なパス（確認）を行い、トーンがプロフェッショナルであり、過度に形式的（またはロボット的）でないことを確認します。
   - 完成したメッセージをユーザーに返し、そのままコピー＆ペーストして送信できるようにします。
